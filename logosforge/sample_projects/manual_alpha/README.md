# LogosForge Alpha Manual Sample Projects

These files are simple **manual testing projects** for LogosForge Desktop
Alpha. They are **not production templates** and are **not app code** — each
is a project file in the app's native interchange format, one per writing
mode. They were generated by the app's own exporter (`export_json`) and
verified loadable by the app's own importer (`validate_import_data` +
`import_json`); no production code was added or changed to create them.

## Format

- **Extension:** `.json` (the format used by **Open Project** and
  **Export → Full Project**).
- **Required keys present in every file:** `project`, `characters`,
  `places`, `notes`, `scenes` (plus `psyke_entries`, `chapters`, `outline`,
  `plot_timeline`, `continuity` as emitted by the real exporter).
- The writing mode is carried by `project.narrative_engine` +
  `project.default_writing_format`.

## Files

| File | Mode | Structure |
|------|------|-----------|
| `alpha_sample_novel.json` | Novel | Act 1 → Chapter 1 → Scene |
| `alpha_sample_screenplay.json` | Screenplay | Act 1 → Chapter 1 → Scene (screenplay body) |
| `alpha_sample_graphic_novel.json` | Graphic Novel | Act 1 → Page 1 → Scene → Panel 1 + Panel 2 |
| `alpha_sample_stage_script.json` | Stage Script | Act 1 → Chapter 1 → Scene (stage body) |
| `alpha_sample_series.json` | Series | Act 1 → Chapter 1 → Scene (see Series note) |

## Each project includes

- At least two characters: **Ada North** and **Milo Voss**.
- At least one PSYKE / Story-Bible element: the **Theme “Threshold”**
  (on load the two characters are also promoted to PSYKE character entries,
  so each project shows 3 PSYKE entries).
- Minimal Manuscript content for its mode.
- Minimal Outline structure (derived from the scene's Act/Chapter labels;
  Graphic Novel derives Act → Page → Scene → Panel from the scene body).
- Two Notes: a per-mode manual-test note and a **Unicode stress note**
  (`"Curly quotes", em dash — ellipsis … emoji 🐕, Zampanò, città, perché`).

## How to open in LogosForge

1. Launch the app (`python3 run.py`).
2. **Projects → Open Project…** (or File → Open) and pick the `.json` file.
   The app imports it as a new project (the original file is never
   overwritten; opening the same file again re-activates that project rather
   than duplicating it).

## Manual checks

For every project:

- Open it; confirm the correct **writing mode**.
- Confirm the **Manuscript** opens and shows the content.
- Confirm the **Outline** opens with the expected structure.
- Confirm **PSYKE** has at least one element (Threshold theme + characters).
- Confirm at least **two characters** (Ada North, Milo Voss).
- **Save / reload** and confirm content persists.
- **Export** (Markdown / TXT / JSON; Fountain for Screenplay) if desired;
  confirm UTF-8 text (incl. the Unicode note) survives.

### Special Graphic Novel check (the renderer fix)

Open `alpha_sample_graphic_novel.json` and confirm:

- **Manuscript** uses the **shared full text/block editor** UX (the
  "Graphic Novel" mode label + word count, ACT / SCENE / PAGE / PANEL
  blocks with editable Visual / Caption / Dialogue / SFX / Notes).
- Manuscript does **NOT** show the old **“Comics Script”** screen, a
  scene dropdown, or page-manager chrome ("Delete Page", "+ Panel" in a
  page header, "+ Add Page").
- **Outline** uses the **shared block/card** UX (Act card → Page block →
  Scene group → Panel cards), not a thin tree or a page-form manager.
- Outline structure is **Act → Page → Scene → Panel** (Chapter is not a
  primary Graphic Novel unit).

## Known format limitations (not bugs)

- **Series Season/Episode hierarchy is not part of the JSON interchange
  format.** The exporter/importer round-trip the canonical flat
  Act → Chapter → Scene (the documented Phase-1 boundary; the Series
  Navigator is the canonical Season/Episode surface). So
  `alpha_sample_series.json` loads as a **Series** project with one
  Act → Chapter → Scene; the Series Navigator starts without Seasons —
  use the Navigator's **Convert to Season/Episode** to build the
  Season → Episode → Act → Chapter → Scene hierarchy in-app.
- **Project writing language is not carried by the JSON interchange format**
  (it lives in per-project settings, not in export/import), so these
  samples do not set a language code; the app defaults to English. This
  does not affect Unicode writing, which the samples exercise via the
  Unicode stress note.
- Graphic Novel **act-wide page start offsets** (`gn_page_start`) are not
  part of the interchange format; the sample uses the default auto-chained
  layout (Scene starts on Page 1).
