# Logosforge HTTP API

The Python core is the **authoritative backend**; this HTTP API exposes it as
stable DTOs and safe actions for a **shared React UI** used by two frontends:

| Frontend            | Transport / environment                                  |
| ------------------- | -------------------------------------------------------- |
| **Electron desktop**| React UI + a locally-launched Python API on `127.0.0.1`. |
| **Web / PWA**       | Same React UI calling a LAN or remote API base URL.      |

The two differ only in *transport*, never in product logic. The API contains no
business logic — every route delegates to existing `logosforge` core services.

---

## Running the server

```bash
pip install -r requirements.txt        # installs fastapi + uvicorn

# Run with environment configuration (defaults to desktop / 127.0.0.1:8765)
python -m logosforge.api

# Or override on the CLI
python -m logosforge.api --host 0.0.0.0 --port 9000 --mode lan --db ./logosforge.db
```

Interactive docs and the OpenAPI schema are auto-generated by FastAPI:

* Swagger UI  → `http://127.0.0.1:8765/docs`
* OpenAPI JSON → `http://127.0.0.1:8765/openapi.json`

Embedding the server programmatically (e.g. from an Electron launcher):

```python
from logosforge.api import create_api, ApiConfig
from logosforge.db import Database

app = create_api(db=Database("logosforge.db"), config=ApiConfig(mode="desktop"))
# hand `app` to uvicorn, or use logosforge.api.server.run(config)
```

## Modes & configuration

Configuration comes from `API_*` environment variables (CLI flags win):

| Variable               | Default       | Meaning                                            |
| ---------------------- | ------------- | -------------------------------------------------- |
| `API_HOST`             | `127.0.0.1`   | Bind address.                                      |
| `API_PORT`             | `8765`        | Bind port.                                         |
| `API_MODE`             | `desktop`     | `desktop` \| `lan` \| `remote`.                    |
| `API_ALLOWED_ORIGINS`  | *(empty)*     | Comma-separated CORS origins (lan/remote).         |
| `API_AUTH_TOKEN`       | *(empty)*     | If set, all `/api/*` routes (except health) require `Authorization: Bearer <token>`. |
| `LOGOSFORGE_DB_PATH` | `logosforge.db` | SQLite project DB the API serves.              |

**CORS / security**

* **desktop** — any `http://localhost:*` / `http://127.0.0.1:*` origin is
  allowed (Electron/Vite pick a free port). Bind to `127.0.0.1` only.
* **lan / remote** — only origins in `API_ALLOWED_ORIGINS` are allowed. Set
  `API_AUTH_TOKEN` before exposing the API beyond localhost. Authentication is
  a single clean hook (`require_auth`) ready to grow into real auth.

## Conventions

* Base URL: **`/api`**. All project data is scoped by `{project_id}` in the path.
* Requests/responses are JSON. `PATCH` bodies are **partial** — only provided
  fields change; omitted fields are preserved.
* Errors use a stable envelope:

  ```json
  { "error": { "code": "not_found", "message": "Scene 5 not found" } }
  ```

  Codes: `not_found` (404), `bad_request` (400), `forbidden` (403),
  `not_implemented` (501), `validation_error` (422), `assistant_error` (502).
* Internal ORM rows are never returned — only the DTOs in
  `logosforge/api/schemas.py`.

## Endpoints

### Health
```
GET  /api/health → { status, service, mode, version, api_version, core_version }
```
`api_version` is the stable HTTP DTO/action **contract** version (also the
`info.version` in `/openapi.json`, used to generate the shared React client).
`core_version` is the Logosforge build (e.g. `0.9.0-alpha`). Clients (Electron
desktop and Web/PWA) read these to verify they're talking to a compatible
backend. `version` mirrors `api_version` for backward compatibility.

### Projects
```
GET    /api/projects
POST   /api/projects                       { title, description?, narrative_engine?, default_writing_format? }
GET    /api/projects/{project_id}
POST   /api/projects/{project_id}/open
POST   /api/projects/{project_id}/save
POST   /api/projects/{project_id}/close
DELETE /api/projects/{project_id}          → 501 (not supported via API)
GET    /api/projects/{project_id}/settings
PATCH  /api/projects/{project_id}/settings { settings: { ... } }   # merged
```

### Scenes / manuscript
```
GET    /api/projects/{project_id}/scenes
POST   /api/projects/{project_id}/scenes
GET    /api/projects/{project_id}/scenes/{scene_id}
PATCH  /api/projects/{project_id}/scenes/{scene_id}
DELETE /api/projects/{project_id}/scenes/{scene_id}
```

### Outline (hierarchical)
```
GET    /api/projects/{project_id}/outline                         # nested tree
POST   /api/projects/{project_id}/outline/nodes
PATCH  /api/projects/{project_id}/outline/nodes/{node_id}
DELETE /api/projects/{project_id}/outline/nodes/{node_id}
```

### Plot (scene-derived; block id = plotline name)
```
GET    /api/projects/{project_id}/plot
PATCH  /api/projects/{project_id}/plot/blocks/{block_id}          { plotline?, color_label? }
```

### Timeline (scene-derived; event id = scene id)
```
GET    /api/projects/{project_id}/timeline
POST   /api/projects/{project_id}/timeline/events
PATCH  /api/projects/{project_id}/timeline/events/{event_id}
```

### PSYKE
```
GET    /api/projects/{project_id}/psyke/entries
POST   /api/projects/{project_id}/psyke/entries
GET    /api/projects/{project_id}/psyke/entries/{entry_id}
PATCH  /api/projects/{project_id}/psyke/entries/{entry_id}
DELETE /api/projects/{project_id}/psyke/entries/{entry_id}
GET    /api/projects/{project_id}/psyke/relations
POST   /api/projects/{project_id}/psyke/relations                 { source_id, target_id, relation_type? }
DELETE /api/projects/{project_id}/psyke/relations/{relation_id}   # relation_id = "{source_id}:{target_id}"
GET    /api/projects/{project_id}/psyke/progressions
POST   /api/projects/{project_id}/psyke/progressions
GET    /api/projects/{project_id}/psyke/search?q=
```

### Notes
```
GET    /api/projects/{project_id}/notes
POST   /api/projects/{project_id}/notes
PATCH  /api/projects/{project_id}/notes/{note_id}
DELETE /api/projects/{project_id}/notes/{note_id}
```

### Assistant & connector
```
POST   /api/projects/{project_id}/assistant/chat        { message, history?, system_prompt? }
POST   /api/projects/{project_id}/assistant/action      { action, args }   # safe action layer
GET    /api/projects/{project_id}/assistant/settings    # api_key never returned
PATCH  /api/projects/{project_id}/assistant/settings
GET    /api/projects/{project_id}/connector/actions     # registered actions + param schema
POST   /api/projects/{project_id}/connector/execute     { action, args }
```

All mutating connector/assistant actions pass through
`logosforge.connector_executor.execute_action`, which validates inputs against
the action's param schema and only permits **registered** actions. Read actions
are always allowed; **write** actions respect the desktop connector settings
(`connector_enabled` / `connector_allow_writes`) so remote writes stay an
explicit opt-in. The API never mutates the DB directly on the assistant's behalf.

### Export
```
POST   /api/projects/{project_id}/export
```
Body (`ExportRequestDTO`):
```json
{ "export_type": "story_elements", "format": "json",
  "include_scenes": true, "include_ids": false, "summaries_only": false }
```
* `export_type`: `story_elements` | `psyke_data` | `full_project`
* `format`: `json` | `markdown` | `csv`
* Response: `payload` (json), `content` (markdown text), or `files`
  (`{filename: csv_text}`). Reuses `logosforge.data_export`.

**Create a scene**
```http
POST /api/projects/1/scenes
{ "title": "Opening", "summary": "It begins", "plotline": "Main", "tags": ["draft"] }
```
```json
{ "id": 7, "title": "Opening", "summary": "It begins", "plotline": "Main",
  "tags": ["draft"], "sort_order": 0, "order_index": 1,
  "character_ids": [], "place_ids": [] }
```

**Assistant chat**
```http
POST /api/projects/1/assistant/chat
{ "message": "Suggest a midpoint twist", "history": [] }
```
```json
{ "reply": "Consider revealing that the mentor...", "cached": false }
```

## Change events (live React sync)

Two transports share the same event names so React handles both identically:

**SSE (preferred)**
```
GET /api/projects/{project_id}/events            # live stream (text/event-stream)
GET /api/projects/{project_id}/events?once=true  # drain buffered events and close
```
Each message:
```
event: scene_changed
data: {"id": 12, "event": "scene_changed", "project_id": 1, "data": {"scene_id": 7}, "ts": 1.7e9}
```

**Polling (fallback)**
```
GET /api/projects/{project_id}/events/poll?since={cursor}
→ { "events": [ ... ], "cursor": 12, "known_events": [ ... ] }
```
Clients keep the last `cursor` and pass it as `since` to get only new events.

**Event names**
```
project_loaded, project_data_changed, scene_changed, scenes_changed,
outline_changed, plot_changed, timeline_changed, psyke_changed,
notes_changed, dashboard_changed, assistant_action_completed
```

The API owns its own Qt-free event broker (`logosforge/api/events.py`); it
does not require the desktop Qt event loop, so the PyQt app and the API can run
independently.

## Desktop vs Web summary

| Aspect        | Desktop / Electron                | Web / PWA                          |
| ------------- | --------------------------------- | ---------------------------------- |
| API location  | bundled, `127.0.0.1:<port>`       | LAN or remote server               |
| `API_MODE`    | `desktop`                         | `lan` / `remote`                   |
| CORS          | localhost origins (regex)         | `API_ALLOWED_ORIGINS` list         |
| Auth          | none needed (loopback)            | set `API_AUTH_TOKEN`               |
| Data          | local SQLite file                 | server-side SQLite                 |
| Live updates  | SSE                               | SSE (or polling fallback)          |
